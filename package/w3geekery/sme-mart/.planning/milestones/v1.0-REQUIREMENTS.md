## v2 Requirements

### Advanced Migration

- **ADV-01**: Dual-write validation mode (parallel write to both Neon + Pipeline for verification)

- **ADV-03**: Notification center migrated from Neon to AuditgraphDB CardService

### Performance

- **PERF-01**: ensureIndexed() polling utility for critical paths where eventual consistency is insufficient
- **PERF-02**: GQL query batching for nested relationship queries

## Out of Scope

| Feature | Reason |
|---------|--------|
| Direct GQL mutations | Platform doesn't support them — writes go through Pipeline only |
| Apollo/URQL client libraries | Platform provides native GQL endpoint; extra abstraction adds no value |
| CDC (Change Data Capture) | Over-engineered for 8 entities at current scale |
| Real-time sync daemon | Pipeline is push-based; no daemon needed |
| Database adapter pattern | Direct swap is simpler; domain services already isolated |
| Component-level changes | Domain service public APIs unchanged — components don't know about migration |
