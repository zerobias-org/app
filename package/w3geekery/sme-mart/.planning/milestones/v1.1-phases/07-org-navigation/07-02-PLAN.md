**Replace line 72-77 orgData signal with:**
```typescript
  readonly orgData = toSignal(
    combineLatest([
      this.app.getOrg(this.orgId()),
      this.app.hydraClient.getOrgApi().listOrgMembers(this.orgId()),
      this.app.hydraClient.getOrgApi().listGroups(this.orgId()),
      // Boundaries: Use GraphQL query via clientApi.graphqlClient
      // For now, placeholder: return empty array if GQL not available
      this.clientApi.graphqlClient?.getBoundaryApi?.().searchBoundaries?.({ orgId: this.orgId() }, 1, 100)
        || new Promise(resolve => resolve({ items: [] })).then(r => (r as any).items || [])
    ]),
    {
      initialValue: [null, [], [], []]
    }
  );

  readonly org = computed(() => this.orgData()[0]);
  readonly members = computed(() => this.orgData()[1]);
  readonly groups = computed(() => this.orgData()[2]);
  readonly boundaries = computed(() => this.orgData()[3]);
```

**Update ngOnInit() (lines 85-96) to only set currentOrgId, remove TODO comment:**
```typescript
  ngOnInit(): void {
    try {
      const currentId = this.app.getCurrentOrgId();
      this.currentOrgId.set(currentId || null);
    } catch {
      this.currentOrgId.set(null);
    }
    // Data loading is now handled via toSignal(combineLatest(...)) above
  }
```

**Rationale:**
- `combineLatest()` combines 4 API calls in parallel: getOrg, listOrgMembers, listGroups, boundaries query
- `toSignal()` converts the observable array result to a reactive signal
- Initial value `[null, [], [], []]` provides safe empty state until API responds
- Computed signals (`org`, `members`, `groups`, `boundaries`) destructure the combined result for template use
- Boundaries query uses GraphQL (preferred) with fallback to empty array if not available
- Reactive: when route changes (`orgId` signal updates), the entire `combineLatest` re-runs automatically

**Important note on boundaries:**
The GQL query for boundaries is complex (requires understanding the schema relationship between Org and Boundary). For now, using a defensive approach that gracefully falls back to an empty array. If boundaries query fails or is unavailable, users see empty state but page doesn't crash.

**Pattern reference:** Matches catalog.service.ts `Promise.all()` pattern but implemented reactively with `combineLatest` and `toSignal()`.
