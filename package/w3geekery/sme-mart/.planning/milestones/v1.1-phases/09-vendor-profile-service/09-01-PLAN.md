```typescript
@Injectable({ providedIn: 'root' })
export class VendorProfileService {
  private readonly pipelineWrite = inject(PipelineWriteService);
  private readonly graphqlRead = inject(GraphqlReadService);

  // ── Query ──
  
  async listProfileItems(orgId: string, section?: SectionType): Promise<MarketplaceProfileItem[]>
  async getProfileItem(id: string): Promise<MarketplaceProfileItem | null>
  
  // ── Mutation ──
  
  async createProfileItem(orgId: string, req: CreateMarketplaceProfileItemRequest): Promise<MarketplaceProfileItem>
  async updateProfileItem(id: string, req: Partial<CreateMarketplaceProfileItemRequest>): Promise<MarketplaceProfileItem>
  async deleteProfileItem(id: string): Promise<void>
  
  // ── Private Helpers ──
  
  private getFields(): string[]
  private fromGql(gql: GqlMarketplaceProfileItemResponse): MarketplaceProfileItem
  private toGql(domain: MarketplaceProfileItem): Record<string, unknown>
  private parseData(dataStr: string): CorporateIdentityData | ...
  private serializeData(data: ...): string
}
```

**Key implementation details:**

1. **listProfileItems(orgId, section?)**: 
   - Call `graphqlRead.query()` with filters: `{ orgId: .eq.{orgId} }` + optional `{ section: .eq.{section} }`
   - Filter out soft-deleted items (check for `dateDeleted` field)
   - Parse JSON `data` field for each item
   - Return sorted list (e.g., by name)

2. **getProfileItem(id)**:
   - Check cache first: `this.pipelineWrite.getCached('MarketplaceProfileItem', id)`
   - If cached, parse and return
   - Otherwise fetch via GQL and cache

3. **createProfileItem(orgId, req)**:
   - Validate request (name required, section valid, data valid)
   - Generate UUID for id
   - Serialize data to JSON string
   - Push to Pipeline via `pushEntity('MarketplaceProfileItem', { id, orgId, section, name, description, data (serialized), expiresAt, status, createdAt, updatedAt })`
   - Return created item (parse data back to domain)

4. **updateProfileItem(id, req)**:
   - Fetch current item (cache or GQL)
   - Merge partial update (shallow merge)
   - Re-serialize data if provided in req
   - Push full object to Pipeline
   - Return updated item

5. **deleteProfileItem(id)**:
   - Fetch current item
   - Set `status: 'deleted'` or add `dateDeleted` timestamp
   - Push to Pipeline with deletion marker

6. **parseData(dataStr)**: 
   - Try `JSON.parse(dataStr)`
   - On error, log warning and return empty object matching section type
   - Return typed object (CorporateIdentityData, etc.)

7. **serializeData(data)**:
   - Return `JSON.stringify(data)`

8. **fromGql(gql)**: 
   - Map GQL camelCase to domain snake_case using mapGqlToNeon
   - Parse `data` field from JSON string to typed object
   - Return domain object

9. **toGql(domain)**:
   - Map domain snake_case to GQL camelCase using mapNeonToGql
   - Serialize `data` typed object to JSON string
   - Return GQL object

10. **getFields()**:
    - Return array of all GQL field names for this entity
    - Use field mapping keys: `[...Object.keys(MARKETPLACE_PROFILE_ITEM_FIELD_MAPPING.neonToGql), 'dateDeleted']`

**Error handling:**
- Throw descriptive errors for validation failures (invalid section, missing name, etc.)
- Log and handle JSON parse errors gracefully
- Use same error patterns as VettingService

**Org scoping (per D-02):**
- Caller passes `orgId` explicitly — service does NOT auto-detect current session org
- This keeps service flexible for Phase 12 (cross-org viewing)

**Per D-01:** Strong typing for section data flows down to UI — no `any` types, all 6 section types have defined interfaces.
