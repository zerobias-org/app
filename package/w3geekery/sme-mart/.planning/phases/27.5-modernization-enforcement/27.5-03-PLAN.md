<objective>
Add a CI lint gate that runs on every PR and on every push (except to deploy-target branches), scoped to **only the files changed in the diff**, reusing Plan 02's `.lintstagedrc.json` so pre-commit and CI behavior are identical. Replace the prior full-repo `test-and-lint.yml` (commit `e901e25`) with a leaner `lint.yml` that requires no auth, no secrets, and no Vault.

**Why diff-based, lint-only, no-auth:**

1. **Diff-based scope** — The codebase has ~1561 pre-existing modernization violations. A full-repo lint would punish every PR for code it didn't touch. Director's enforcement model is "touch it = own it"; pre-existing untouched files do not block PRs.
2. **Lint-only (no test, no build)** — ENF-05 mandates lint-as-CI-gate. Tests run via developer pre-push; CI build gate is deferred to BACKLOG `CI-BUILD-GATE-1` (blocked on `.env.local` injection + errata-015 secret-in-tracked-file). Adding tests/build here adds scope without satisfying ENF-05 better.
3. **No auth** — `npm ci` against the full lockfile would require private-registry credentials (`@zerobias-org` via `pkg.zerobias.org`, `@zerobias-com` via `npm.pkg.github.com`). Provisioning either Vault role allowlists (cross-team blocker on Brian/Kevin) or fork-side PAT secrets adds complexity for what is fundamentally a lint gate. We install only the public lint tooling we need (eslint + angular-eslint plugins + typescript + lint-staged) directly from the public npm registry with `npm install --no-save`.

**Output:** `.github/workflows/lint.yml` (new), `.github/workflows/test-and-lint.yml` removed, two CI runs proven (one fail, one pass) and recorded in SUMMARY.

**Already-committed work to handle:** Commit `e901e25` on `poc/sme-mart` introduced `test-and-lint.yml` against the prior (full-repo) spec. gsd-execute decides whether to revert + re-author or rewrite atop it; either path is acceptable provided the final tree matches this plan's spec and `test-and-lint.yml` is removed.
</objective>
