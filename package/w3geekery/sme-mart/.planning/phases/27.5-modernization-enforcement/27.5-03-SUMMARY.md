## Plan deviations and why

Two follow-up commits adjusted the workflow during execution:

1. **`e0a3769`** — Plan 03's install list specified `@typescript-eslint/parser` and `@typescript-eslint/eslint-plugin` as separate packages, but this repo's `eslint.config.js` imports the v8 meta package (`import tseslint from 'typescript-eslint'`) plus `@eslint/js`. Without these, ESLint fails at config-load time, not at the lint rule — which the plan's own acceptance criterion forbids. Fix: install the meta packages with versions pinned to `package.json`'s devDeps.

2. **`eb7adf3`** — Plan 03 directed `npm install --no-save` against the workspace cwd. Workspace `.npmrc` maps `@zerobias-org` -> `pkg.zerobias.org` and `@zerobias-com` -> `npm.pkg.github.com`, both auth-required. Even with explicit-package install + `--no-save`, npm reads `package.json` + lockfile and tries to resolve transitive private deps -> 401 on `@zerobias-org/util-connector`. Fix: install lint tooling into `${RUNNER_TEMP}/lint-tools` with a fresh minimal `package.json` (no zerobias deps), then symlink the resulting `node_modules` into the workspace so `eslint.config.js` ESM imports resolve normally. Preserves the auth-free invariant.

Both deviations preserve the plan's success criteria: diff-based scope, lint-only, no auth, no secrets, no Vault.
