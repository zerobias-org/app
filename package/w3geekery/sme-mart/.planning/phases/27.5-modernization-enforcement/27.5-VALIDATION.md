# Phase 27.5 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution. This phase is unusual: the deliverable IS a validation gate. Most checks are exit-code smoke tests, not unit tests. The "test that the rule fires" pattern dominates.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | ESLint (lint exit codes) + bash smoke tests for hook/CI verification + existing Karma (`npm test`) for regression coverage |
| **Config file** | `eslint.config.js` (created in Wave 1), `.husky/pre-commit` (created in Wave 1), `.lintstagedrc.json` (created in Wave 1), `.github/workflows/*.yml` (modified in Wave 1) |
| **Quick run command** | `npm run lint` (after Wave 1 lands the script) |
| **Full suite command** | `npm run lint && npm test` (lint + existing 1537 specs to prove no regression) |
| **Estimated runtime** | ~30s lint + ~3-5min test suite |

---

## Sampling Rate

- **After every task commit:** Run `npm run lint` (where the script exists post-Wave-1).
- **After every plan wave:** Run `npm run lint && npm test` (full lint + regression).
- **Before `/gsd:verify-work`:** Full suite must be green AND all 6 smoke-test scenarios in "Per-Task Verification Map" below must pass.
- **Max feedback latency:** 60s for lint; 5min for full suite.
