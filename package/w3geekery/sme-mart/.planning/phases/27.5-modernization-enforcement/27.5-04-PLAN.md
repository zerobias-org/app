```bash

```

If `npm run lint` does not accept `--format json` directly, invoke ESLint via `npx`:


2. Verify the snapshot is non-empty and well-formed:

```bash

# Expected: a positive integer (number of files inspected).
```

3. Tabulate per-rule counts with `jq`. Save the rollup to a temp file for paste into the audit:

```bash
jq -r '
  [.[] | .messages[]?]
  | group_by(.ruleId)
  | map({rule: .[0].ruleId, count: length})
  | sort_by(-.count)
  | .[]
  | "| \(.rule // "unknown") | \(.count) |"
' /tmp/lint-snapshot.json > /tmp/per-rule.tsv
cat /tmp/per-rule.tsv
```

4. Compute total violations:

```bash
jq '[.[] | .messages[]?] | length' /tmp/lint-snapshot.json
```

5. Tabulate top-10 violating files per rule. For each rule with count > 0:

```bash
RULE="<rule-name>"
jq -r --arg rule "$RULE" '
  [.[] | select(.messages[]?.ruleId == $rule) | {file: .filePath, count: ([.messages[] | select(.ruleId == $rule)] | length)}]
  | sort_by(-.count)
  | .[0:10]
  | .[]
  | "| \(.file | sub("^.*sme-mart/"; "")) | \(.count) |"
' /tmp/lint-snapshot.json
```

Repeat for each rule from the per-rule rollup (typically 4–7 rules; see Plan 01's `eslint.config.js`).

6. Compute total LOC affected — count distinct (file, line) violation sites:

```bash
jq '[.[] | {file: .filePath, line: (.messages[]?.line // 0)}] | unique | length' /tmp/lint-snapshot.json
```

(Note: this counts distinct violation **sites**, not file LOC. Sites is the load-bearing number for cleanup planning. The "total LOC affected" header in the audit refers to "lines that contain at least one violation.")
