**Gathered:** 2026-04-30
**Status:** Ready for planning

<domain>
## Phase Boundary

Install a machine-enforcement gate (ESLint flat config + husky/lint-staged pre-commit + CI lint step) for the Angular 21 modernization rules already documented in CLAUDE.md, MODERNIZATION_GUIDE.md, and Director handoff blocks. After this phase closes, violations of the modernization rules physically cannot land via commit or merge.

This phase **installs the gate**. It does **not** sweep pre-existing violations. Pre-existing violations are annotated with `eslint-disable-next-line <rule>` + a reason comment and tracked under a single BACKLOG entry `MODERN-CLEANUP-1` for a separate cleanup phase.

In scope:
- ESLint flat config (`eslint.config.js`) covering 7 modernization patterns
- `npm run lint` script + `angular.json` lint builder
- husky v9 pre-commit hook running `lint-staged` on staged files only
- CI lint step (in `.github/workflows/deploy.yml` and any other PR-running workflow), running BEFORE the build step
- `INITIAL-AUDIT.md` capturing pre-existing violation counts
- `MODERN-CLEANUP-1` BACKLOG entry
- Documentation updates (CLAUDE.md, MODERNIZATION_GUIDE.md) referencing the new gate

Out of scope:
- Repo-wide modernization sweep
- Prettier / formatting rules
- Stylelint for SCSS
- Test coverage thresholds
- Project-specific custom lint rules beyond the 7 modernization patterns
- `provideAnimationsAsync` deprecation migration (separate dependency-update tracker)

</domain>

<decisions>
## Implementation Decisions (locked from brief)

### Rule Set (7 modernization patterns)

The gate must enforce all 7 patterns. Rule **names** below are the brief's working list — **plan author MUST verify each rule name against the installed `angular-eslint` plugin's actual exports before encoding**. If a rule does not exist under the listed name in the installed version, surface as a plan-time question, do NOT silently substitute.

| # | Pattern enforced | Working rule name (verify before encoding) | Plugin |
|---|---|---|---|
| 1 | Standalone components only; bans NgModule-wrapped components | `@angular-eslint/prefer-standalone` | angular-eslint |
| 2 | Field-level `inject()`; bans constructor parameter DI | `@angular-eslint/prefer-inject` | angular-eslint |
| 3 | Signal-based `input()` / `output()`; bans `@Input()` / `@Output()` decorators | `@angular-eslint/prefer-signals` (verify exact rule name + scope) | angular-eslint |
| 4 | `@if` / `@for` / `@switch`; bans `*ngIf` / `*ngFor` / `*ngSwitch` | `@angular-eslint/template/prefer-control-flow` | angular-eslint template |
| 5 | Bans `import { CommonModule } from '@angular/common'` in component sources | `no-restricted-imports` (custom config) | typescript-eslint base |
| 6 | Bans `<mat-spinner>` deprecated alias; require `<mat-progress-spinner>` | `@angular-eslint/template/no-deprecated-element` OR custom selector rule (verify cleanest path) | angular-eslint template |
| 7 | Bans `@Input()` / `@Output()` decorators if pattern 3's rule doesn't already do it cleanly | `no-restricted-syntax` AST query | typescript-eslint base |

### Configuration scope

- Component sources: `src/app/**/*.{ts,html}` covered by full rule set.
- Test files (`*.spec.ts`): exempted from `prefer-standalone` only (test harness modules sometimes legitimately need NgModule). All other rules apply equally.
- Excluded entirely: `node_modules`, `dist`, `.angular`, generated SDK files (e.g., `MarketplaceProfileItem` and other `@auditmation`/`@auditlogic` generated clients).

### Build & script wiring

- `package.json` script: `"lint": "eslint . --max-warnings=0"` — zero-warning policy. Warnings ARE failures. Non-negotiable.
- `angular.json` lint builder: configured so `ng lint` works for IDE integration. Output must match `npm run lint` exactly.

### Pre-commit hook

- husky v9+ installed.
- `.husky/pre-commit` runs `npx lint-staged`.
- `.lintstagedrc.json` (or `package.json` `"lint-staged"`): `{ "*.{ts,html}": "eslint --max-warnings=0" }` — staged files only, NOT full repo. Hook latency above ~3s breaks Clark's commit cadence.
- The hook runs on every `git commit`. Violations block the commit with the rule name + offending line in stderr.
- Bypass `git commit --no-verify` documented in the SUMMARY but flagged as human-emergency only. Agents MUST NOT use it.

### CI gate

- Add `lint` step to `.github/workflows/deploy.yml` and any other CI workflow running on PRs / pushes.
- Step runs `npm run lint`. Failure fails the workflow. PRs with violations cannot merge.
- The lint step runs BEFORE the build step. Fast-fail; lint is faster than tsc/build.

### Pre-existing violation handling

- After config is wired, run `npm run lint` once and capture output to `.planning/phases/27.5-modernization-enforcement/INITIAL-AUDIT.md`.
- For every existing violation:
  - Apply `eslint-disable-next-line <rule-name>` with one-line comment: `// pre-existing — pre-Phase-27.5 enforcement; track in MODERN-CLEANUP-1`
  - OR fix in-place if trivial (e.g., a single `*ngIf` -> `@if` swap). Do NOT do trivial fixes if they balloon the patch.
- Do NOT do a sweeping repo-wide migration. Scope is the gate, not the cleanup.
- File a single BACKLOG entry `MODERN-CLEANUP-1` capturing the count + category of disable comments left, sized for a future dedicated cleanup phase or rolling-as-touched cleanup.

### Documentation updates

- `CLAUDE.md` "Angular 21 Patterns" section: paragraph noting modernization rules are now machine-enforced via ESLint + pre-commit + CI. Reference the eslint config path. Note that the rules-block in handoff prompts is now redundant with enforcement BUT should be retained in handoffs as belt-and-suspenders + so plan authors don't write code that the gate will reject.
- `.planning/docs/MODERNIZATION_GUIDE.md`: cross-link to the eslint config + add a "If lint fires on you" troubleshooting section with most common violations and their fixes.

### Plan-author non-negotiables (paste verbatim into gsd-execute handoff)

- **Source of truth for rule names:** `angular-eslint`'s installed-plugin docs (`node_modules/@angular-eslint/eslint-plugin/docs/rules/`) + the GitHub release at the version tag matching what gets installed. NOT this brief's exact strings, NOT prior memory entries, NOT general training data. Verify before encoding.
- **Do NOT do a repo-wide migration sweep.** Annotate with `eslint-disable-next-line` for pre-existing violations. Cleanup is a separate phase tracked under `MODERN-CLEANUP-1`.
- **Pre-commit hook MUST run on staged files only**, not the whole repo. Hook latency above ~3s breaks Clark's commit cadence.
- **`--max-warnings=0` everywhere.** Warnings ARE failures. The whole point is to make the gate physical, not negotiable.
- **CI lint step runs BEFORE build step.** Fast-fail; don't burn CI minutes on a build that lint would have rejected.
- **The bypass `git commit --no-verify` is human-authorized emergency only.** Document it but make clear agents must NOT use it.
- **Per-task atomic commits.** Standard discipline restored — one task per commit, per Phase 27 Wave 3 pattern.
