Output: .husky/pre-commit hook, .lintstagedrc.json config, smoke tests verifying rejection + acceptance flows.
</objective>

1. Create .lintstagedrc.json at the project root (same level as package.json, not nested).

2. Content (exact, no modifications):
```json
{
  "lint-staged": {
    "*.{ts,html}": "eslint --max-warnings=0"
  }
}
```

3. Verify JSON is valid:
```bash
jq . .lintstagedrc.json
```

4. This config tells lint-staged:
   - Match any staged file with .ts or .html extension
   - Run eslint --max-warnings=0 on those files
   - Block the commit if eslint exits non-zero

Note: lint-staged automatically stages only the files it lints (does not run against whole repo).

5. Alternative: this config can also be placed in package.json as a top-level "lint-staged" object instead of a separate file. For this plan, use the separate .lintstagedrc.json file.
  </action>
  <acceptance_criteria>

    - JSON is valid: jq . .lintstagedrc.json succeeds
    - Grep confirms "lint-staged" key exists
    - Grep confirms "*.{ts,html}" glob pattern
    - Grep confirms "eslint --max-warnings=0" command
  </acceptance_criteria>
  <automated>jq . .lintstagedrc.json && grep -q 'eslint --max-warnings=0' .lintstagedrc.json</automated>
</task>
