## Known Limitations

- **Pattern 6 (mat-spinner ban):** Deferred to future phase. Angular ESLint template plugin v21.3.1 does not expose a generic `no-restricted-syntax` rule for templates. Possible alternatives: (a) upgrade to newer angular-eslint version with the rule, (b) create project-specific lint rule, (c) rely on code review + MODERNIZATION_GUIDE.md documentation. Recommend (c) for now — mat-spinner is rarely used in modern Angular apps.

- **Parsing errors on .spec.ts files not in tsconfig.app.json:** ESLint reports "parserOptions.project" errors on spec files because tsconfig.app.json excludes them. This is expected ESLint behavior for TypeScript projects. Spec files are still linted (non-TS rules apply); the parser errors don't block the run.
