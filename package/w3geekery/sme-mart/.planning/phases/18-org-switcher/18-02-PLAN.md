# Plan 18.2: Hotfix — Empty Submenu + Placement (Errata 013)

**Phase:** 18 — Org Switcher
**Requirements:** OS-01 (placement), OS-02 (SDK call), OS-04 (populated list filtered correctly)
**Effort:** 2–4 hours
**Depends on:** 18-01 (landed)
**Origin:** Errata 013 — Phase 18 ships non-functional (empty list + buried placement)

## Goal

Fix two post-execute defects in Phase 18 so the org switcher actually works end-to-end:

1. **Empty submenu.** `OrgSwitcherService.loadOrgs()` calls `app.getOrgs()` which returns an internal Subject that usually doesn't emit. Swap it for the method used by `/orgs`: `clientApi.danaClient.getMeApi().listMyOrgs()`. Convert to async/await.
2. **Mispositioned submenu.** The "Switch Organization" trigger is currently buried below Settings. Move it above `My Organizations`, or make the current-org header itself a chevron trigger — whichever pattern the zb/ui portal user-menu uses. Read that component first; don't guess.
3. **Test hardening.** Specs must mock the real SDK path, include an empty-array regression test, and the Playwright E2E must run against a real UAT session asserting populated list + correct placement.

Do not close Phase 18 until UAT screenshot shows a populated list at the correct position.

```typescript
}
```

**Steps:**

1. Read `src/app/pages/orgs/org-list.component.ts:104-135` to confirm the exact async/await shape.
2. Update `OrgSwitcherService.loadOrgs()`:

   - Convert to `private async loadOrgs(): Promise<void>` with try/catch.
   - Call `const orgs = await this.clientApi.danaClient.getMeApi().listMyOrgs();`
   - Populate `rawOrgs` signal with `orgs || []` (treat `undefined`/`null` as empty).

3. Call `loadOrgs()` in constructor (same pattern as `org-list.component.ts`). Keep the existing computed `orgs$` filter + sort chain unchanged.
