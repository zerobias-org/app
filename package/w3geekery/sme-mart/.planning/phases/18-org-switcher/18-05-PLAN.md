# Plan 18.5: Port `zb-ui-resource-image` CSS to Global Styles (Errata 016)

**Phase:** 18 — Org Switcher
**Requirements:** OS-01 (populated visible submenu), OS-03 (current-org distinction works once rows are sane)
**Effort:** 20–30 minutes
**Depends on:** 18-01, 18-02, 18-03, 18-04 (all landed)
**Origin:** Errata 016 — Plan 18-04 markup references `zb-ui-resource-image.s20` classes that live in `zb-ui-lib` (not `ngx-library`); classes resolve to nothing in SME Mart → 14687×1558 Auditmation logo renders at natural size, blows out 400px panel

## Goal

Port the `zb-ui-resource-image` + size-modifier CSS block from `zb-ui-lib` into SME Mart's global `src/styles.scss` so the markup Plan 18-04 already shipped has the styles it needs. No markup changes. No component swaps. Root-cause fix, smallest possible diff.

When SME Mart eventually merges into zb/ui, the ported block gets dropped and the zb-ui-lib originals take over seamlessly.

## Why this and not zb-avatar-label

Considered and rejected per brief. Plan 18-04's markup is correct — only the CSS is missing. Porting the classes:
- Addresses the actual root cause (not routes around it)
- Reusable: `zb-ui-resource-image` + `s*` modifiers are used 13+ places in zb/ui for any resource logo (orgs, products, connections, etc.) — SME Mart will need this pattern again
- Smallest diff: no component edits, no spec changes, no asset changes, no imports

## Files Touched

- `src/styles.scss` (APPEND) — ~50 lines of ported CSS
- `src/app/shared/components/user-profile-dropdown/user-profile-dropdown.component.scss` (AUDIT; MODIFY only if workaround rules exist)

## Tasks

### Task 1: Port `zb-ui-resource-image` CSS to `src/styles.scss`

**Addresses:** Errata 016 root cause
**Type:** style
**TDD:** false (pure CSS port; verification is DevTools + UAT)

**Reference source:** `~/Projects/zb/ui/projects/zb-ui-lib/src/lib/components/components.scss:2085-2193`

**Steps:**

1. Read the brief one more time for the exact CSS block.
2. Append to `src/styles.scss` (at the end is fine; or under an existing "global utilities" section if one exists):

   ```scss
   // =============================================================================
   // Resource image sizing classes — ported from zb-ui-lib components.scss:2085-2193
   // Used wherever <img> renders a resource logo/avatar at a constrained size.
   // See .planning/director/errata/016-org-switcher-image-sizing-regression.md
   // =============================================================================

   img.zb-ui-product-logo {
     max-height: 24px;
     max-width: 24px;
     width: auto;
     height: auto;
     vertical-align: middle;
   }

   img.zb-ui-resource-image {
     max-width: 24px;
     min-width: 24px;
     max-height: 24px;
     width: 24px;
     height: 24px;
     object-fit: contain;
     vertical-align: middle;

     &.s16 {
       max-width: 16px; min-width: 16px;
       max-height: 16px; height: 16px; width: 16px;
       object-fit: contain; vertical-align: middle;
       &.maintain-aspect { min-width: unset; width: auto; max-width: 100%; }
     }

     &.s20 {
       max-width: 20px; min-width: 20px;
       max-height: 20px; height: 20px; width: 20px;
       object-fit: contain; vertical-align: middle;
       &.maintain-aspect { min-width: unset; width: auto; max-width: 100%; }
     }

     &.s32 {
       max-width: 32px; min-width: 32px;
       max-height: 32px; height: 32px; width: 32px;
       object-fit: contain; vertical-align: middle;
       &.maintain-aspect { min-width: unset; width: auto; max-width: 100%; }
     }

     &.float.left {
       float: left;
       &.margin, &.margin4 { margin-right: 4px; }
       &.margin8 { margin-right: 8px; }
       &.margin16 { margin-right: 16px; }
     }
   }
   ```

3. **Explicitly OMIT:**
   - Dark-theme `filter: invert(...)` blocks — commented out in source anyway; defer until SME Mart lands dark theme.
   - `.using-default` rules — not used by Plan 18-04; add later if needed.
   - Plain `img.resource-image` (without the `zb-ui-` prefix) — not used anywhere in SME Mart; scope minimal.

4. Add a section comment at the top of the ported block citing the source file and the errata (included in the scaffold above).

- `grep "filter: invert" src/styles.scss` returns 0 matches.
- `npm run build` (or `ng build --configuration development`) succeeds.
- `npm run lint` clean for `src/styles.scss`.

**Commit:** `style(global): port zb-ui-resource-image CSS from zb-ui-lib (errata 016)`
