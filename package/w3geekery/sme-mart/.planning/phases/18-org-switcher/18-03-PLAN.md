## Tasks

### Task 1: Strip filters from OrgSwitcherService + prune spec

**Addresses:** OS-04 (empty-list residue from over-filtering)
**Type:** refactor + test
**TDD:** false (straight deletion; existing retained tests guard behavior)

**Steps:**

1. Edit `src/app/core/services/org-switcher.service.ts`:
   - Replace the body of the `orgs$` computed with exactly:
     ```typescript
     readonly orgs$ = computed(() => {
       return this.rawOrgs().sort((a, b) => a.name.localeCompare(b.name));
     });
     ```
   - Delete the `SYSTEM_ORG_ID` constant (line ~14).
   - Delete the private methods `isHiddenOrg`, `isSystemOrg`, `isOpsOrg` (lines ~86-107) in full — the spec only references `isOpsOrg` in an inline comment, not as a callable.
   - Leave `loadOrgs()`, `switchTo()`, signals, and imports intact.
   - Resulting file should be ~70 lines.

2. Edit `src/app/core/services/org-switcher.service.spec.ts`:
   - Delete the entire `describe('Filter Rules', ...)` block (lines 71–171 in the current file) — three `it(...)` tests inside.
   - Inside `describe('orgs$ signal', ...)`, keep the sort assertion; if the fixture in that test leans on "filters out hidden/system", simplify it to just verify `localeCompare` sort order on a 3-item array (include no hidden/system/ops entries — all three should survive sort).
   - Delete the inline comment `// Currently isOpsOrg returns false, so all orgs pass through` if it remains after the above.
   - `describe('loadOrgs regression tests', ...)` (empty-array + error) stays verbatim.
   - `describe('switchTo', ...)` stays verbatim (5 tests: no-op, open dialog, selectOrg call, success path, error path).
   - Remove any now-unused imports, fixture builders, or mock helpers that only the Filter Rules tests used.

3. Run targeted spec:
   ```bash
   npm test -- --include='**/org-switcher.service.spec.ts' --watch=false
   ```
   All remaining tests pass (expected count ~8: 2 loadOrgs + 1 orgs$ + 5 switchTo).

**Self-check before commit:**
