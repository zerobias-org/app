## Architecture Reference

zb/ui portal pattern (`organization-switcher.component.html:22`):

```html
<img
  [src]="org.avatarUrl | staticImageUrl"
  class="zb-ui-resource-image s20"
  imgDefault
  [default]="'./assets/unknown-company.svg'"
/>
```

Pieces from `@zerobias-org/ngx-library`:
- `ZbStaticImageUrlPipe` — resolves relative/static URLs
- `ZbImgDefaultDirective` — swaps to fallback on image load error
- `zb-ui-resource-image` / `s20` classes — 20px square framing

zb/ui uses plain `<img>` + fallback directive (NOT `ZbAvatarLabelComponent`) for org rows — keep it simple.

## Risks

- **ngx-library directive not compatible with standalone imports** — `ZbImgDefaultDirective` is exported from the library (confirmed in public-api.ts per director brief). If it's packaged as a module rather than standalone, wrap usage or re-check public-api. Mitigation: Task 2 verifies compile cleanly before proceeding.
- **`staticImageUrl` pipe transforms URL in an unexpected way** — if `org.avatarUrl` is already absolute, the pipe may no-op; if undefined, may emit empty string (and `imgDefault` handles the error). Mitigation: `imgDefault` fallback is the safety net; Task 5 test covers the no-avatar case.
- **Vendored SVG diverges from source over time** — if zb/ui updates `unknown-company.svg`, SME Mart copy drifts. Mitigation: acceptable trade-off (asset is a generic placeholder); revisit only if zb/ui overhauls branding.
- **Dead SCSS rules from previous circle-dot pattern** — easy to miss one. Mitigation: Task 4 explicitly lists `.current-marker` and `.spacer` removal; `rg "current-marker|spacer" src/app/shared/components/user-profile-dropdown/` after Task 4 should return no hits.
