Create `src/app/core/services/demo-visibility.service.ts` as a singleton service (`providedIn: 'root'`).

**STRATEGY LOCK (per CONTEXT.md "Object.tag Field Pattern" — REVISED 2026-05-01, Option X):**
This service implements **client-side post-filtering**. Server-side `tag` negation (`.ne.` / `.not in.`) is non-viable per Decision-Probe-1 (FAIL 2026-05-01). The service does NOT build any GQL filter object. Instead it owns a pure tag-membership predicate plus a generic post-filter that consumers wrap their fetched results with.

The previous `buildExcludeFilter()` API is **eliminated** — it was designed to produce a server-side filter that does not work.

**Service surface (LOCKED):**

1. Inject `ProjectContextService` via field-level `inject(ProjectContextService)`.

2. **`isLocalDemoTagged(record: { tag?: TagShape | null }): boolean`** — pure predicate over a record's `tag` field.
   - Returns `true` iff any element of the record's `tag` array carries a `value` matching EITHER demo UUID from `DEMO_TAG_UUID_LIST` (Task 1).
   - Returns `false` for `tag: null`, `tag: undefined`, `tag: []`, or arrays whose elements all carry non-demo UUIDs.
   - `TagShape` is the array form ZB GQL returns: `Array<{ value: string }>` (matches the ingest payload from `onboarding-bootstrap.service.ts:179`). If the project already has a shared `Tag` type, import and reuse it; otherwise define an inline minimal `{ value: string }` shape.
   - PURE function — no `inject()` reads inside, no signal reads. Take only the record argument; return only the boolean. (Lets it be called from non-DI contexts and unit-tested without TestBed.)

3. **`applyVisibility<T extends { tag?: TagShape | null }>(records: T[]): T[]`** — post-filter for arrays of fetched records.
   - If `projectContext.isAdmin()` returns `true` → return `records` unchanged (admin bypass).
   - Otherwise → `records.filter(r => !this.isLocalDemoTagged(r))`.
   - Generic `<T>` so callers preserve their domain types (Engagement[], Bid[], Note[], etc.) without casting.
   - Reads `isAdmin()` exactly once per call — no caching, no signal subscription. Each call respects the current admin signal value.

4. JSDoc on each public method explaining: (a) what it filters, (b) the admin short-circuit, (c) why this is client-side (cite Decision-Probe-1).

**What this service does NOT do:**
- Build any `Record<string, string>` filter for GraphqlReadService (that path is dead).
- Mutate input arrays — return new array via `Array.prototype.filter`.
- Re-call the admin SDK directly — only reads `ProjectContextService.isAdmin()`.

**Imports:**
- `DEMO_TAG_UUIDS`, `DEMO_TAG_UUID_LIST` from the constants module (Task 1).
- `ProjectContextService` from `src/app/core/services/project-context.service.ts`.
- `inject` from `@angular/core`.

## Security Considerations

| Threat | Category | Component | Disposition | Mitigation |
|--------|----------|-----------|-------------|-----------|
| Demo-tag UUID exfiltration | Information Disclosure | Constant values | Accept | UUIDs are constants, not secrets. Code visibility (public constant) is acceptable. UUIDs have no inherent security value. |
| Admin signal spoofing | Elevation of Privilege (EoP) | ProjectContextService | Mitigate | Filter-building service reads `ProjectContextService.isAdmin()`, which is hydrated ONLY by onboardingGuard via verified platform SDK call. Service does NOT re-call admin SDK. Prevents per-session admin flag forgery. |
| Filter bypass (client-side filter only) | Authorization | DemoVisibilityService | Transfer | This Wave 0 service builds client-side filters. GQL API acceptance of the filter (server-side enforcement) is Wave 2's responsibility (GraphqlReadService augmentation). Wave 0 accepts the architectural split: client builds the filter, server enforces. No mitigation at this layer; responsibility shifts downstream. |

## Risk Assessment

- **Low:** UUIDs as constants (no credential risk)
- **Low-Medium:** Admin signal read-only (no local state mutation possible; signal set once by guard)
- **Deferred to Wave 2:** Server-side filter enforcement (not in scope for constants/helper service)

</threat_model>
