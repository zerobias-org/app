<objective>
Create NoteFolderService to migrate NoteFolder hierarchy management from Neon SmeMartDbService to AuditgraphDB Pipeline+GraphQL. The service implements a flat-fetch + client-side tree rebuild pattern: query all NoteFolders for an engagement in one GraphQL call, rebuild the parent-child tree locally using parentId mapping. This avoids N+1 queries and recursive GQL fetching.

Purpose: Notes can be organized in folder hierarchies with unrestricted depth. The tree structure is preserved through migration and queryable without expensive recursive operations.

Output: NoteFolderService fully migrated, tree rebuild algorithm implemented and tested, zero component changes required.
</objective>

  <action>
    1. Create new NoteFolderService with @Injectable({ providedIn: 'root' })

    2. Constructor: Inject PipelineWriteService, GraphqlReadService, ImpersonationService (for userId)

    3. Implement createFolder(engagementId: string, data: CreateNoteFolderRequest): Promise<NoteFolder>
       - Build gqlData with camelCase: id (UUID), engagementId, parentId (nullable), name, sortOrder, color, createdAt, updatedAt
       - Map gqlData → Neon type using mapGqlToNeon<NoteFolder>(gqlData, NOTE_FOLDER_FIELD_MAPPING.gqlToNeon)
       - Fire-and-forget pipelineWrite.pushEntity('NoteFolder', gqlData)
       - Return optimistically immediately

    4. Implement getNoteFolderTree(engagementId: string): Promise<NoteFolderTreeNode[]>
       - THIS IS THE CORE ALGORITHM (flat-fetch + rebuild):
         a. Query GraphQL: graphqlRead.query<GqlNoteFolderResponse>('NoteFolder', ['id', 'engagementId', 'parentId', 'name', 'sortOrder', 'createdAt'], { filters: { engagementId: `.eq.${engagementId}` }, pageNumber: 1, pageSize: 1000 })
         b. Map all results to Neon NoteFolder type using NOTE_FOLDER_FIELD_MAPPING.gqlToNeon
         c. Build in-memory tree:
            - Create allFolders array from mapped results
            - Create folderMap: Map<id, NoteFolder> for O(1) lookups
            - Filter rootFolders = allFolders.filter(f => !f.parent_id)
            - Recursive buildNode(parent: NoteFolder): NoteFolderTreeNode function that:
              * Returns parent with children array
              * Children = allFolders.filter(f => f.parent_id === parent.id).sort((a,b) => (a.sort_order ?? 0) - (b.sort_order ?? 0))
              * Each child recursively calls buildNode (tree recursion)
            - Return rootFolders.sort(...).map(buildNode)
         d. CYCLE DETECTION (safety): Track visited Set<id> during tree build, abort if parentId already visited (log error, skip child)

    5. Implement updateFolder(folderId: string, data: UpdateNoteFolderRequest): Promise<NoteFolder>
       - Build gqlData with updated fields (name, parentId, sortOrder, color, updatedAt)
       - Map to Neon type, optimistic return, fire-and-forget push

    6. Implement deleteFolder(folderId: string): Promise<void>
       - Soft-delete: set archived: true, push to pipeline, optimistic return
       - OR hard delete if preferred — confirm with existing codebase pattern

    7. Implement signal for reactive tree state (optional):
       - readonly folderTree = signal<NoteFolderTreeNode[]>([])
       - setFolderTree(engagementId, tree) method if needed by components

    8. Error handling:
       - Cycle detection: if visited.has(folderId) during tree build, log error and exclude subtree (graceful degradation)
       - Missing folders: if query returns empty, return [] (no error)

    Coverage: Addresses MIG-11 (write NoteFolder via Pipeline), MIG-12 (read via GQL), MIG-13 (hierarchy preserved via tree rebuild)
  </action>
