**BidsService pattern:**
- Uses `inject(PipelineWriteService)` and `inject(GraphqlReadService)`
- `pipelineWrite.push('Bid', bidObject)` for writes
- `graphqlRead.getById<GqlBidResponse>('Bid', id, fieldsList)` for reads
- Field mappings handle snake_case ↔ camelCase conversion
- Error messages thrown as strings (not error objects)

**SmeMartProjectService pattern:**
- Uses same Pipeline + GQL pattern
- Model interface in models/sme-mart-project.model.ts
- Field mapping constants in field-mappings.ts
- GQL types in gql-types.ts

**RfpInvitation status values (per decision D-02):**
- pending: Buyer sent invitation, vendor has not responded
- accepted: Vendor accepted invitation (can bid)
- declined: Vendor declined invitation (cannot bid)
- revoked: Buyer revoked invitation after sending
- expired: Auto-set when RFP closed with pending invitations
- requested: Vendor requested invitation (awaiting buyer approval)

**Gate validation rules (per decision D-12, FLAG-2):**
- If project.isInvitationOnly = false, gate passes (open bidding)
- If project.isInvitationOnly = true:
  - PASS: invitation exists AND status = 'accepted'
  - FAIL: no invitation found → "You do not have an invitation to submit a bid on this RFP"
  - FAIL: status = 'pending' → "Your invitation is pending. Acceptance required before bidding."
  - FAIL: status = 'declined' → "You declined this invitation and cannot bid."
  - FAIL: status = 'revoked' → "Your invitation was revoked and is no longer valid."
  - FAIL: status = 'expired' → "Your invitation has expired."
  - FAIL: status = 'requested' → "Your invitation request has not been approved yet."
