## UI Component Patterns (from existing codebase)

**"My X" pages (reference: my-project-list.component.ts):**
- Card/table layout with status filters
- Empty state component
- Search/filter input with debounce
- Pagination or infinite scroll

**Tab pattern (reference: project-detail.component.ts):**
- MORE_TAB_GROUPS for grouping tabs (e.g., Governance group)
- Tab route synchronization via @Input() activeTab
- Tab content lazy-loaded or conditionally rendered

**ngx-library components to use:**
- ZbSimpleAutocompleteComponent (vendor org search)
- ZbResourceStatusComponent (status badges for invitation statuses)
- ZbEmptyStateContainerComponent (empty state messaging)
- Material: mat-tab-group, mat-dialog, mat-button, mat-chip, mat-card

**Angular 21 patterns in this project:**
- Control flow: `@if (condition) { } @else { }` (NOT *ngIf)
- Loops: `@for (item of items; track item.id) { }` (NOT *ngFor)
- Signals: `signal()`, `computed()`, `toSignal()` for reactive state
- inject() function (NOT constructor)
- Standalone components with imports array

**Status chip colors (from ZB UI neverfail-lib):**
- pending: light gray #e9e9e9
- accepted: light green #d8ecba
- declined: light red #eed5d1
- revoked: light red #eed5d1
- expired: light gray #e9e9e9
- requested: light blue #d7e0ee (awaiting approval)
