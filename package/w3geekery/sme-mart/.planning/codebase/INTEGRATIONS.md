# External Integrations

**Analysis Date:** 2026-03-18

## APIs & External Services

**ZeroBias Platform:**
- ZeroBias Portal (authentication, org selection, login redirect)
  - SDK: `@zerobias-com/zerobias-angular-client` (wraps zerobias-client)
  - Auth: API key-based (`ZEROBIAS_*_API_KEY` env vars), session inheritance in iframe mode
  - Endpoints: Platform at `https://[environment].zerobias.com/api`
  - Purpose: User authentication, organization context, platform initialization

**ZeroBias GraphQL API:**
- Custom schema class queries (Engagement, Bid, BidResponse, ServiceOffering, Note, NoteFolder, Review, SmeMartDocument)
  - SDK: `@zerobias-com/graphql-sdk` (ExecuteRawGraphqlQuery)
  - Client: `ZerobiasClientApi.graphqlClient.getBoundaryApi()`
  - Boundary: Platform boundary (ID: `2842fab1-ceff-4ec4-bf09-ce5e7c33c3e2`)
  - Purpose: Read SME Mart entities from AuditgraphDB
  - Service: `src/app/core/services/graphql-read.service.ts`

**ZeroBias Pipeline API (AuditgraphDB Write):**
- Receiver pipeline for differential updates
  - SDK: `@zerobias-com/platform-sdk` (SimpleBatch, Pipeline.receive)
  - Pipeline ID: `091d5068-0527-4f45-9839-37f6d5c1669e` (production)
  - Purpose: Push/upsert SME Mart entities into AuditgraphDB
  - Service: `src/app/core/services/pipeline-write.service.ts`
  - Class IDs: Engagement, Bid, BidResponse, ServiceOffering, Note, NoteFolder, Review, SmeMartDocument

**ZeroBias Catalog APIs:**
- Role, Skill, Framework, Segment, Service Segment, Product lookups
  - SDK: `ZerobiasClientApi.platformClient` (CatalogRoleApi, SegmentApi, FrameworkApi)
  - Purpose: Fetch reference data for UI dropdowns (cached per session)
  - Service: `src/app/core/services/catalog.service.ts`

**ZeroBias Hydra APIs:**
- Tag management (create, search, list)
  - SDK: `ZerobiasClientApi.hydraClient.getTagApi()`
  - Purpose: Create and search ZB platform tags, assign to resources
  - Service: `src/app/core/services/sme-mart-tag.service.ts`, `src/app/core/services/engagement-hierarchy.service.ts`

**ZeroBias Hub Module (Generic SQL):**
- DataProducer interface for Neon database access
  - SDK: `@zerobias-org/data-utils` (DataProducerClient)
  - Connection: `e3c874f5-5fd8-4fbc-8120-19861e28b19e` (QA, created 2026-02-11)
  - Module: `@auditlogic/module-auditmation-generic-sql` (v0.5.0)
  - Purpose: Browse/query/update Neon tables via DataProducer abstraction (production path)
  - Service: `src/app/core/services/sme-mart-db.service.ts` (mode: 'hub')
  - Fallback: Direct Neon HTTP when Hub unavailable

## Authentication & Identity

**Auth Provider:**
- ZeroBias Platform (custom)
  - Implementation: OAuth-like redirect to `/login` (handled by platform)
  - Session: Inherited from parent portal (iframe mode) or API key-based (standalone)
  - Flow:
    1. App calls `ZerobiasClientApp.init()`
    2. If no session, redirects to `/login` on ZeroBias platform
    3. User authenticates
    4. Platform redirects back to app
    5. Session persists in localStorage/sessionStorage

**Headers & Cookies:**
- Request header: `Authorization: APIKey {API_KEY}` (dev/CI mode via proxy)
- Request header: `Dana-Org-Id: {ORG_ID}` (SDK, multi-tenancy)
- Cookie: `dana-org-id` (injected by proxy, fallback if header missing)

**Environment Variables:**
- ZEROBIAS_UAT_API_KEY / ZEROBIAS_UAT_ORG_ID (UAT)
- API_KEY / ZB_ORG_ID (CI, deprecated)
- ZEROBIAS_QA_API_KEY / ZEROBIAS_QA_ORG_ID (QA)
- ZEROBIAS_PROD_API_KEY / ZEROBIAS_PROD_ORG_ID (Production)

## Cross-Cutting Platform Integration

**Org Selection & Multi-Tenancy:**
- SDK manages org selection via `ZerobiasClientOrgIdService`
- Selected org cached in `sessionStorage` with key `zb-current-dana-org-id`
- SDK injects `Dana-Org-Id` header on all platform API calls
- Proxy injects `dana-org-id` cookie as fallback

**Session Management:**
- Handled entirely by ZeroBias platform
- No manual refresh/renewal logic needed
- Session expiry managed by platform policies

**Socket/WebSocket:**
- Disabled for SME Mart (socketUrlPath: '' in all environment configs)
- Portal-only feature, not needed for standalone app

**Proxy Middleware (Development):**
- Intercepts `/api/*` requests
- Rewrites to ZeroBias platform target (`https://[env].zerobias.com/api`)
- Injects Authorization header (API key)
- Injects/updates Dana-Org-Id cookie
- Error handling: Logs proxy errors, returns 502 on failure

## Data Flow Summary

```
┌─────────────────────┐
│   SME Mart App      │
│   (Angular 21)      │
└──────────┬──────────┘
           │
    ┌──────┴──────┐
    │             │
    ▼             ▼
 ZeroBias    Neon/Hub
 Platform    Database
 (Auth,      (Entity
  Catalog,    Storage)
  Tags,
  GraphQL,
  Pipeline)
```

1. **Auth**: App → Platform (init, session check, redirect to `/login`)
2. **Catalog**: App → Platform API (roles, skills, frameworks, segments)
3. **Tags**: App → Platform Hydra API (create, search, assign)
4. **Read**: App → GraphQL API (query entities from AuditgraphDB)
5. **Write**: App → Pipeline API (push entities to AuditgraphDB)
6. **Storage**: App → Neon (direct HTTP or via Hub DataProducer)
