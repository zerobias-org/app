# environment.neon.ts was committed with live credentials and then wiped by gsd-executor during Phase 18

## Symptom

Impersonation users list in SME Mart rendered empty starting 2026-04-15 (post Phase 18 Plan 18-01 merge). Diagnosed 2026-04-16: `SmeMartDbService.connectNeon()` found `environment.neonConnectionString === ''`, never connected, `neonQueryPublic` threw, `ImpersonationService.loadUsers()` swallowed the error in a `console.warn`, list stayed empty.

## Root cause

Commit `ac8e994` ("feat(org-switcher): implement OrgSwitcherService with filtering and switchTo flow" — Phase 18 Plan 18-01, 2026-04-15) modified `src/environments/environment.neon.ts`:

```diff
-export const NEON_DATABASE_URL = 'postgresql://neondb_owner:npg_NjsYRTy2U6re@ep-aged-fog-af9wu771.c-2.us-west-2.aws.neon.tech/neondb?sslmode=require';
+export const NEON_DATABASE_URL = '';
```

The file's own header says `AUTO-GENERATED by scripts/gen-neon-env.mjs — DO NOT COMMIT`. It is produced by the `predev` npm hook which reads `process.env.NEON_DATABASE_URL` (from `.env.local` via dotenv-cli) and writes the value into the file. When the env var is unset, the script writes an empty string.

During Phase 18 execution the gsd-executor subagent ran a build/serve command. Its shell did not have `.env.local` loaded (no dotenv wrapper around the agent's bash calls, and no NEON_DATABASE_URL in the parent shell env). The `predev` hook fired, gen script wrote empty string, and the subsequent `git add` / `git commit` in the executor's atomic-commit workflow picked up the diff along with the actual org-switcher code changes.

## Historical context

This only happened *now* because of an older, separate problem: commit `68abe4d` (2026-02-24 — "fix(sme-mart): commit environment.neon.ts stub for Vercel builds") originally committed the file **with a real production Neon connection string in it**, despite the DO-NOT-COMMIT header. That put the credential in `origin` and in every developer's clone, meaning:

1. Local dev "worked" with no `.env.local` because the URL was hardcoded in the committed file
2. Any `predev` run in an environment without the env var would silently erase the credential
3. Developers had no visible indication the env var was needed

So the Feb 2026 credential-in-git decision masked a latent failure mode that finally triggered under Phase 18 execution.

## Impact

- **Feature regression:** impersonation unusable in local dev from 2026-04-15 until fix.
- **Security:** `postgresql://neondb_owner:npg_NjsYRTy2U6re@ep-aged-fog-af9wu771...` was live on `origin/poc/sme-mart` and all derivative branches from 2026-02-24 until 2026-04-15. The credential is in git history permanently on all clones. Rotation advised even though the wipe commit removed it from current HEAD — the history copy is still there.
- **Process signal:** gsd-executor's atomic-commit sweep included a file marked DO NOT COMMIT. Same class as errata 007 (uncommitted drift), errata 009 (director touched GSD artifact), errata 010 (silent stubs) — executor discipline on "which files to stage" needs tightening.

## Fix (this session)

1. **Created `.env.local`** (gitignored) with `NEON_DATABASE_URL=postgresql://...@ep-aged-fog-af9wu771-pooler...neondb` from Neon MCP.
2. **Ran `npx dotenv-cli -e .env.local -- node scripts/gen-neon-env.mjs`** — wrote the URL back into `environment.neon.ts`. File is in working tree but unstaged (MODIFIED flag per git status). Do NOT commit this regeneration.

## Remediation (needs follow-up action)

### Immediate
- [ ] **Rotate the exposed Neon credential.** The password `npg_NjsYRTy2U6re` was on `origin` from 2026-02-24 to 2026-04-15. Treat as compromised. Generate new Neon role or reset password via Neon console, update `.env.local` locally, notify any other developers who may have this credential in their shells / clones.
- [ ] **Do NOT stage `environment.neon.ts`** — the regeneration this session was only to restore local functionality. `git restore --staged` if it appears, or just leave it unstaged forever.

### Structural
- [ ] **Remove `environment.neon.ts` from the committed tree entirely.** File shouldn't be in git at all per its own header. Replace with a `.gitkeep` or checked-in stub with empty URL + instructions, and put the real file in `.gitignore`. Vercel builds should get the URL via Vercel env vars, not a committed file.
- [ ] **Add a runtime check in gen-neon-env.mjs** — if `NEON_DATABASE_URL` is empty AND the output file currently has a non-empty URL, either abort or write a `// regeneration-skipped` comment instead of wiping the URL. Prevents accidental credential erasure by future env-missing runs.
- [ ] **Update `.env.local.example` to include `NEON_DATABASE_URL` placeholder.** Current example lists ZB API keys only; Neon URL was silently expected because of the committed stub.

### Process (gsd-executor discipline)
- [ ] **Executor should refuse to stage files marked `DO NOT COMMIT` in their header.** Cheap pre-commit check: `grep -l "DO NOT COMMIT" $(git diff --cached --name-only)` should block the commit.
- [ ] **Executor pre-dispatch check for commands that trigger hooks.** When executor runs `npm run dev` or any command with `pre*` hooks, those hooks may produce diffs unrelated to the task. Either run commands in a clean env and discard incidental diffs, or fail loud if pre-hook-generated files appear in staging.

## Prevention patterns for WATCH-LIST

- [ ] **Committed file header says "DO NOT COMMIT"** — BLOCK. Files with this header must be gitignored and absent from `git ls-files`. If the header exists and the file is tracked, file errata and fix both.
- [ ] **Executor stages auto-generated files** — BLOCK. If `git diff --cached` includes a file matching `AUTO-GENERATED|DO NOT COMMIT|GENERATED BY`, pause and require explicit human confirmation.
- [ ] **Credential-shaped strings (`postgresql://`, `npg_*`, API keys) in committed files** — BLOCK. Secret-scanner on pre-commit hook should catch these regardless of the file's intent.
- [ ] **`predev` / `prestart` npm hooks that regenerate config from env vars** — FLAG. Either make the generator idempotent (preserve existing value when env is empty) or ensure the hook output is gitignored. Silent credential-wiping is the worst-of-both-worlds default.
