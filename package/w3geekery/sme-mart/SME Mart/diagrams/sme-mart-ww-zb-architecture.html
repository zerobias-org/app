ZeroBias + Work Worlds + SME Mart

three-layer architecture — system of record → execution → marketplace

ZeroBias is the system of record (tasks, findings, boundaries, auth). Work Worlds is the execution and performance layer (missions, scoring, proof of work). SME Mart is the marketplace (discovery, RFPs, contracting, engagement center, assessments, credential verification). Both WW and SME Mart are third-party apps built on the ZeroBias platform using ZeroBias SDKs.

Key Architecture Decisions

Engagement Center

SME Mart owns it. The marketplace/business view of engagements — status, contracting, proposals, timeline. Work Worlds provides a separate execution view of the same underlying ZeroBias Tasks.

WW Mission Events in Timeline

SME Mart's timeline displays read-only events from Work Worlds:

  • Mission Started
  • Mission Completed (score, duration, outcome)
  • Assessment Passed
  • Score Updated
Scheduling

SME Mart builds scheduling so buyers and providers coordinate when work happens. A scheduled block in SME Mart can trigger a WW mission. Scheduling = marketplace coordination; execution = WW.

Shared Identity

ZeroBias user ID is the shared key across all three systems. Provider profiles in SME Mart, scoring data in WW, and tasks in ZB all link via the same userId.

Data Source Mapping

SME Mart Feature Data Source Notes
Provider score / reputation Work Worlds API Display-only. WW computes from mission performance.
Credential verification (Credly) SME Mart SME Mart owns Credly integration. Badges displayed on provider profiles.
Adaptive assessments SME Mart SME Mart owns provider vetting quizzes. Adaptive difficulty, scoring.
Provider matching for RFPs Work Worlds API WW matchmaker uses skills + performance history.
Engagement task tracking ZeroBias Task API Both apps read/write the same ZB Tasks.
RFPs, proposals, contracting SME Mart DB Purely SME Mart's domain. Neon PostgreSQL.
Provider profiles & services SME Mart DB SME Mart owns marketplace profiles.
Timeline activity SME Mart + WW Events Comments & status changes from SME Mart; mission events from WW.
Scheduling SME Mart Marketplace-side coordination. Can trigger WW missions.

Build vs. Consume

Consume from WW APIs
  • Scoring logic or algorithms
  • Matching algorithm
  • Mission execution & proof of work

These are Work Worlds features. Display-only in SME Mart.

Build in SME Mart
  • Marketplace discovery & search
  • RFPs, proposals, contracting
  • Engagement center (business view)
  • Adaptive assessments (provider vetting)
  • Credential verification (Credly)
  • Timeline with WW event placeholders
  • Scheduling feature
