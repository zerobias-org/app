Core Principle: Transparency Center is a FUNCTION

Kevin: "Nothing exists that's 'both' — each party owns their piece. The function produces the finished result."

Zero default visibility. Each party's internal data is invisible to all others unless explicitly permitted. The Transparency Center reads from both sides and renders a unified view — it stores nothing itself.

Per-Task "Transparency Controls" Panel

Brian: "Each task has a Transparency Controls section that publishes to the entangling bridge."

This panel appears on EVERY task (demand or supply side). It controls what gets published to the transparency function. Think of it as the "share settings" for this specific task's data.

The toggle model. Each data point on a task is independently publishable. Private notes are NEVER publishable (no toggle). Attachments and assignee are private by default. Description, due date, and status default to published (for transparency requirements).
